# AI 简历 ATS 匹配诊断工作台视觉语言规范

## 使用方式

后续让 Codex 生成页面时，可以把下面这句放在最前面：

```text
请根据以下视觉语言规范，生成一个高保真的 AI简历匹配诊断工作台页面。不要做 landing page，直接做产品主界面。
```

## 项目视觉方向

这是一个面向求职者的 AI 简历润色与 ATS 匹配分析网站。整体视觉应是“专业、可信、数据化、清晰”的 SaaS 工作台，而不是营销型 landing page。

首屏就是可用的产品界面，重点呈现简历与 JD 的匹配诊断、ATS 通过率、关键词缺口和优化建议。

## 核心气质

- 像一款严肃的职业工具，而不是花哨的 AI 玩具。
- 让用户感觉自己的简历、隐私和职业机会被认真对待。
- 信息密度较高，但要干净、有秩序、容易扫描。
- 视觉上参考 Teal + Jobscan 的结合：现代 SaaS 工作台 + 数据诊断报告。

## 页面结构

- 使用完整桌面 Web App 布局，不要做传统营销首页。
- 顶部是紧凑导航栏：产品名、当前项目、导出、保存、账户入口。
- 主区域采用三栏工作台布局：
  - 左栏：上传与输入区域，包括 Resume、Job Description、目标公司、目标岗位。
  - 中栏：核心诊断报告，包括匹配度大分数、ATS readiness、分项评分、关键词覆盖。
  - 右栏：AI 优化建议，包括缺失关键词、推荐改写、优先级任务、生成优化简历按钮。
- 主按钮文案类似 “Generate optimized resume” 或 “生成优化简历”。
- 页面不需要大标题 hero，不需要插画，不需要复杂背景装饰。

## 色彩规范

- 主背景：`#F7F9FC` 或 `#F8FAFC`。
- 面板背景：`#FFFFFF`。
- 主文本：`#111827`。
- 次级文本：`#6B7280`。
- 边框：`#E5E7EB` 或 `#DDE3EA`。
- 主色：深蓝 `#1E3A8A` 或 `#1D4ED8`，用于主要按钮、导航选中态、关键链接。
- 成功和匹配：墨绿色或 emerald `#059669` / `#10B981`。
- 风险和待优化：琥珀色 `#D97706` / `#F59E0B`。
- 错误和严重缺失：克制使用红色 `#DC2626`。
- 不要使用大面积紫色渐变、霓虹色、玻璃拟态、彩色光斑、装饰性圆球背景。

## 排版规范

- 使用现代无衬线字体：Inter、Geist、SF Pro 或 `system-ui`。
- 字体不要跟随视口宽度缩放。
- 字间距保持正常，不使用负 `letter-spacing`。
- 标题克制：
  - 页面级标题：24-30px。
  - 面板标题：14-18px。
  - 数据数字可以更大，例如匹配度 44-56px。
- 文案简洁，避免大段解释性文字。
- UI 内不要写“如何使用本功能”的长说明，重点靠布局和控件表达。

## 组件风格

- 面板使用 1px 浅边框、轻微阴影、白底。
- 圆角不超过 8px。
- 不要卡片套卡片。
- 按钮高度 36-44px，主按钮深蓝底白字。
- 次级按钮白底、浅边框、深色文字。
- 使用 lucide 风格线性图标，比如 Upload、FileText、Target、CheckCircle、AlertTriangle、Download、Sparkles。
- 图标只作为辅助，不要用复杂插画。
- 标签 chip 用于关键词：
  - 已匹配关键词：浅绿色底 + 深绿色字。
  - 缺失关键词：浅琥珀底 + 深琥珀字。
  - 低相关关键词：浅灰底 + 灰色字。

## 数据可视化

- 核心区域需要一个醒目的匹配度模块，例如圆环评分 78%。
- 分项评分用横向进度条：
  - Skills Match
  - Experience Relevance
  - Keyword Coverage
  - ATS Formatting
  - Seniority Fit
- 每个分项有状态色：绿色代表良好，琥珀色代表待优化，红色只用于严重问题。
- 右侧建议按优先级排列：
  - High priority
  - Medium priority
  - Quick wins
- 每条建议应有简短标题、影响说明和操作按钮，例如 Apply、Review、Ignore。

## 交互状态

- 上传区域要有明确的拖拽态、已上传态和解析中状态。
- 分析中可以显示骨架屏或进度状态，但不要花哨动画。
- 生成优化简历后，建议可以变成 “Applied” 状态。
- 错误提示要短、具体、靠近问题区域。
- Hover 状态使用轻微背景变化或边框加深，不要夸张动效。

## 版式细节

- 页面最大宽度可以接近全屏，适合作为工作台。
- 主内容区使用 24px 左右外边距。
- 栏间距 16-24px。
- 左栏约 25%，中栏约 45%，右栏约 30%。
- 面板内部 padding 16-24px。
- 保持视觉密度，但每组信息之间要有清晰分隔。
- 桌面端优先，移动端可以改为单列堆叠：上传、匹配报告、优化建议。

## 不要做

- 不要做营销 landing page。
- 不要做大面积渐变背景。
- 不要使用紫色 AI SaaS 风格。
- 不要使用可爱插画、卡通人物、夸张 3D 图形。
- 不要把所有内容塞进一个巨大卡片。
- 不要让页面看起来像简历模板网站，而要像“简历诊断与优化工作台”。

## 建议页面命名

- Resume Match Report
- ATS Optimization Dashboard
- AI Resume Diagnostic Workspace

## 目标效果

生成出来的 UI 应该像一个已经可以上线的 B2C SaaS 产品工作台。用户第一眼能看到：

1. 我上传了什么简历和 JD；
2. 当前匹配度是多少；
3. 哪些关键词缺失；
4. 哪些部分需要优化；
5. 点击哪个按钮可以生成更匹配岗位的简历。
